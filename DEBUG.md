# LumiJ - デバッグガイド

## 概要
このガイドでは、すべてのハードウェアを接続しなくても、LumiJシステムをテスト・検証できる組み込みデバッグ機能の使い方を説明します。

## 前提条件
- M5StackライブラリがインストールされたArduino IDE
- シリアルモニタ（ボーレート115200）
- 各M5Stackデバイス（Stamp1、Dial、Stamp2）

## デバッグ機能

### 1. シリアルモニタデバッグ
すべてのデバイスはUSBシリアル経由で詳細なデバッグ情報を出力します。
各デバイスをPCに接続し、115200bpsでシリアルモニタを開いてください。

### 2. シミュレーションモード（Dialのみ）
Dialにはシミュレーションモードがあり、Dial単体とシリアルコマンドだけでシステム全体をテストできます。

## シミュレーションコマンド

### 基本コマンド
HELP                - 使用可能なコマンド一覧表示
STATUS              - 現在の状態を表示

### 入力シミュレーション
SIM_KEY:<id>        - キーマトリクス入力をシミュレート（0〜15）
SIM_KEYBUTTON       - キーボタン押下（BPMモード切替）
SIM_TAP:<interval>  - タップ入力（ミリ秒指定）

## テスト手順

### 1. 基本動作テスト
1. DialをPCに接続
2. シリアルモニタを開く（115200bps）
3. HELP を送信
4. STATUS を送信

### 2. キーマトリクスのテスト
SIM_KEY:3 → ID=3確認
SIM_KEY:7 → ID=7確認

### 3. BPMモードテスト
SIM_KEYBUTTON → BPMモード
SIM_TAP:500 ×3 → BPM=120
SIM_KEYBUTTON → 通常モード

## デバッグ出力例

### 通常動作
Dial: UI Controller Initialized
SIM_KEY:5
Key selected: ID=5

### BPMモード
BPMモード開始
タップからBPM計算
BPM: 120

## トラブルシューティング

### よくある問題
- 反応しない → ボーレート確認
- BPM動かない → モード確認
- 通信不良 → 配線確認

## チェックリスト
- 初期化ログ表示
- HELP/STATUS動作
- キー操作反映
- BPM計算OK

## 応用デバッグ
SIM_KEY:0 → SIM_KEYBUTTON → SIM_TAP → STATUS

## 統合テスト
全デバイス接続して動作確認
